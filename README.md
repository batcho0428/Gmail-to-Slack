# Gmail-to-Slack

Gmailで受信・送信したメールを自動的にSlackチャンネルに転送するGoogle Apps Scriptです。

## 概要

- Gmail の受信箱と送信済みから1時間以内のメールを検出
- メールをSlackへ自動転送
- スレッド単位でSlackのスレッド機能に対応
- 処理済みメールの管理（重複送信なし）
- 5分ごとの自動実行

## 事前準備

### 1. Slackアプリの設定

1. https://api.slack.com/apps にアクセス
2. 「Create New App」→「From scratch」でアプリを作成
3. 左メニュー「OAuth & Permissions」を開く
4. Scopesの「Bot Token Scopes」に「chat:write」を追加
5. 「Install to Workspace」をクリック
6. 「Bot User OAuth Token」（`xoxb-...` で始まる）をコピー

### 1-2. 作成したBotをSlackチャンネルに追加

1. **チャンネルを開く**: 転送先のSlackチャンネルを表示
2. **メンバーを追加**: チャンネル名の右側にある「⋯」→「メンバーを追加」または「インテグレーション」などから、作成したアプリを招待
   - または、チャンネルの詳細パネルから「アプリ」セクションで直接追加
3. **チャンネルID を取得**: チャンネル詳細（チャンネル名右上の「ℹ」アイコン）から、チャンネルID（`C` で始まる英数字）を確認

### 2. スプレッドシートの準備

- 新規スプレッドシートを作成
- スプレッドシートIDを控える（URLの `/d/` と `/edit` の間の文字列）

## セットアップ手順

### ステップ 1: スクリプトプロパティの初期化＆スプレッドシート自動作成＆トリガー設定

GASエディタで以下の手順を実行します：

1. **GASエディタで `setupAll()` 関数を実行**
   - 画面上部の「実行」ボタンを押すか、`setupAll` 関数を選択してから実行
   - 初期セットアップ（スクリプトプロパティの初期化、スプレッドシート作成、トリガー設定）が一度に実行されます
   - コンソールの実行ログを確認して完了を確認

### ステップ 2: スクリプトプロパティの値を正しい値に更新

1. GASエディタ左メニューの「プロジェクトの設定」（歯車アイコン）を開く
2. 一番下の「スクリプト プロパティ」セクションで以下のキーの値を更新：
   - **SLACK_BOT_TOKEN**: ステップ 1-6 で取得したBot Token (`xoxb-...`)
   - **SLACK_CHANNEL_ID**: ステップ 1-8 で取得したチャンネルID (`C...`)
   - その他 SPREADSHEET_ID, SHEET_NAME は `setupAll()` で自動設定済み

### ステップ 3: 検索条件の確認（オプション）

Code.gs 内の `SEARCH_QUERY` 変数を必要に応じて調整：

```javascript
// 【デフォルト】受信箱と送信済みの両方
const SEARCH_QUERY = '{in:inbox in:sent} newer_than:1h';

// 【受信メールのみ】
// const SEARCH_QUERY = 'in:inbox newer_than:1h';
```

## 使用方法

### 自動実行

- `setupAll()` 実行後、main関数が5分ごとに自動実行されます
- GASエディタ → 「トリガー」（時計アイコン）メニューで自動実行の設定を確認できます

### 手動実行

- GASエディタで main関数を選択し「実行」ボタンを押すと、即座に実行できます

## メッセージの色分け

転送されたメールは、以下のルールで色分けされて表示されます：

| メール種類 | 色 | 説明 |
|----------|-----|------|
| **受信メール（新規）** | 🟢 濃い緑 (#2f7d32) | Gmailの受信箱から受け取った新しいメール |
| **受信メール（返信）** | 🔵 濃い青 (#1f5fbf) | 既存のメールスレッドに対する返信 |
| **送信メール（新規）** | 🟩 薄い緑 (#a5d6a7) | Gmailから自分が送信した新しいメール |
| **送信メール（返信）** | 🔷 薄い青 (#9ecbff) | 既存のメールスレッドに対する自分からの返信 |

**補足**:
- 濃い色（新規）と薄い色（返信）で視覚的に区別できます
- スレッド返信は、Slackのスレッド機能を使用して、元のメール下に"返信"として表示されます

## 設定ファイル

### スクリプトプロパティ

| キー | 説明 | 例 |
|------|------|-----|
| SLACK_BOT_TOKEN | Slack Bot Token | `xoxb-1234567890...` |
| SLACK_CHANNEL_ID | 転送先SlackチャンネルのID | `C1234567890` |
| SPREADSHEET_ID | 処理状況を管理するスプレッドシートのID | `1ABCDefGHIJ...` |
| SHEET_NAME | 使用するシート名 | `MailLists` |

### スプレッドシート

自動作成されるスプレッドシート内：
- **A列**: Message ID（Gmailメッセージの一意識別子）
- **B列**: Gmail Thread ID（スレッドの識別子）
- **C列**: Slack ts（Slack投稿のタイムスタンプ）

## トラブルシューティング

### setupAll() 実行後、メールが転送されない

1. スクリプトプロパティが正しく設定されているか確認
2. SLACK_BOT_TOKEN と SLACK_CHANNEL_ID に正しい値が入っているか確認
3. Slackチャンネルにアプリが招待されているか確認
4. GASエディタのトリガー設定を確認（左メニュー → 「トリガー」）

### エラーメッセージが出る

- コンソールを確認して詳細なエラーメッセージを確認
- スプレッドシートが正しく作成されているか確認
- ネットワーク接続を確認
